Naming Conventions & Folder Structure


    
    

      
📁 Required Folder Structure

      
Each plugin must live in its own subfolder named exactly like the plugin. The bat file scans for MCG_* folders and expects this exact layout:

      

        
C:\CustomTools\Inventor\

        
├── Install_AutoLoadInventorAddin.bat

        
│

        
├── MCG_InventorSymbolHandler\  ← folder name = plugin name

        
├── MCG_InventorSymbolHandler.addin  ← same name

        
└── MCG_InventorSymbolHandler.dll   ← same name

        
│

        
├── MCG_InventorDrawingPlugin\

        
├── MCG_InventorDrawingPlugin.addin

        
└── MCG_InventorDrawingPlugin.dll

        
│

        
└── MCG_AnotherPlugin\

        
├── MCG_AnotherPlugin.addin

        
└── MCG_AnotherPlugin.dll

      

      

        ⚠️ Critical: The folder name, .addin filename, and .dll filename must be 100% identical. If any one of the three differs, the bat will skip that plugin entirely.
      

    


    
    

      

        
📝 File & Folder Naming Rules

        

          

            RULE 1
            

              
MCG_ prefix is mandatory

              
Must start with MCG_ (case-sensitive). The bat uses MCG_* wildcard to discover plugins automatically.

            

          

          

            RULE 2
            

              
Folder = .addin = .dll

              
All three must share the exact same name. Example: folder MCG_MyTool → files MCG_MyTool.addin + MCG_MyTool.dll

            

          

          

            RULE 3
            

              
No spaces

              
Use underscores only. Spaces in folder/file names will break the bat script and prevent Inventor from loading the add-in.

            

          

          

            RULE 4
            

              
Assembly path is relative

              
The <Assembly> tag in the .addin file uses only the filename — no full path. Inventor resolves it from the folder where the .addin lives.

            

          

          

            RULE 5
            

              
Unique GUID per plugin

              
Each plugin must have its own ClassId and ClientId. Never reuse a GUID from another plugin. Generate with PowerShell: [guid]::NewGuid()

            

          

        

      


      

        
✅ Valid vs ❌ Invalid Examples

        

          

            
✅ VALID

            

              MCG_InventorSymbolHandler\

                MCG_InventorSymbolHandler.addin

                MCG_InventorSymbolHandler.dll
            

          

          

            
❌ INVALID — folder name doesn't match

            

              InventorSymbolHandler\

                MCG_InventorSymbolHandler.addin

                MCG_InventorSymbolHandler.dll
            

          

          

            
❌ INVALID — name mismatch between files

            

              MCG_SymbolHandler\

                MCG_InventorSymbolHandler.addin

                MCG_SymbolHandler.dll
            

          

          

            
❌ INVALID — spaces in name

            

              MCG_Symbol Handler\

                MCG_Symbol Handler.addin

                MCG_Symbol Handler.dll
            

          

        

      

    


    
    

      
⚙️ How the Inventor BAT File Works

      
The bat automates copying plugins from your source folder to Inventor's official Addins directory. Here's what it does step by step:


      


        

          

            
1

            

          

          

            
Define source & target paths

            
Source: C:\CustomTools\Inventor\  →  Target: %APPDATA%\Autodesk\Inventor 2023\Addins\

          

        


        

          

            
2

            

          

          

            
Check source exists

            
If C:\CustomTools\Inventor\ is not found → prints [ERROR] and exits immediately.

          

        


        

          

            
3

            

          

          

            
Create Addins directory if missing

            
Auto-creates %APPDATA%\Autodesk\Inventor 2023\Addins\ if it doesn't exist yet (e.g. first-time install).

          

        


        

          

            
4

            

          

          

            
Scan all MCG_* subfolders

            
Loops through every subfolder starting with MCG_. For each folder, it checks that both FolderName.addin and FolderName.dll exist inside.

            

              Both exist → proceed to copy
              Missing .dll → skip + warn
              Missing .addin → skip + warn
            

          

        


        

          

            
5

            

          

          

            
Create matching subfolder in Addins & copy files

            
Creates ...\Addins\MCG_PluginName\ at the target, then copies both .addin and .dll into it. If the folder already exists, it overwrites the files (safe update).

          

        


        

          

            
6

          

          

            
Print summary & prompt restart

            
Reports how many plugins were found and how many were successfully activated. Always prompts user to restart Inventor 2023 to apply changes.

          

        


      


      

        💡 Adding a new plugin? Just drop its MCG_NewPlugin\ subfolder (with matching .addin + .dll) into C:\CustomTools\Inventor\ and re-run the bat — no other changes needed.
      

    


  


  
  

    
Add-in File Template (.addin)

    

      
XML Structure Overview

      
The .addin file is a manifest that Inventor uses to discover and load your plugin.

      

        

          ClassId & ClientId
          Unique GUIDs generated for your project
        

        

          Assembly
          Absolute path to C:\CustomTools\Inventor\MCG_PluginName.dll
        

        

          LoadOnStartUp
          Set to "1" for automatic loading
        

        

          UserUnloadable
          Set to "1" to allow user to unload via Add-Ins dialog
        

        

          SupportedSoftwareVersionGreaterThan
          26.. = Inventor 2023 (v27.x) and above
        

      

      

<?xml version="1.0" encoding="utf-8"?>
<!--
  File is copied automatically to:
  %APPDATA%\Autodesk\Inventor 2023\Addins\
-->
<Addin Type="Standard">

  <ClassId>{YOUR_GUID_HERE}</ClassId>
  <ClientId>{YOUR_GUID_HERE}</ClientId>

  <DisplayName>MCG_PluginName</DisplayName>
  <Description>Short description of what this plugin does</Description>

  <Assembly>MCG_PluginName.dll</Assembly>

  <!-- Inventor 2023 = version 27.x, support from 26.x -->
  <SupportedSoftwareVersionGreaterThan>26..</SupportedSoftwareVersionGreaterThan>

  <LoadOnStartUp>1</LoadOnStartUp>

  <UserUnloadable>1</UserUnloadable>

  <Hidden>0</Hidden>

</Addin>
      

      
💡 Replace MCG_PluginName in DisplayName and Assembly. The Assembly path is relative — the .addin and .dll must be in the same folder. Generate a fresh GUID for ClassId & ClientId.

    

  


  
  

    
Professionalizing the Add-in Experience

    

      

        
01

        
Ribbon Menu Setup (UI/UX)

        
Create a dedicated MCG TOOLS ribbon to keep standard menus clean.

        
    
          
  • Simple Groups: Organize tools into Model, Drawing, and Utility groups.
    • 
          
  • Smart View: Only show tools relevant to the current file type (3D model or drawing).
    • 
        

      

      

        
02

        
One Central Folder

        
Consolidate all tools under C:\CustomTools\Inventor for easier updates.

        
    
          
  • Fast Updates: Scripts automatically pull newest files from the server.
    • 
          
  • One Version: Everyone uses the same tools simultaneously.
    • 
        

      

      

        
03

        
Standard Look & Feel

        
Consistent design builds trust across the toolset.

        
    
          
  • Icons: High-quality images with clear backgrounds. (16px & 32px .png or .ico)
    • 
          
  • Help Text: Short tooltip first, expand on hover.
    • 
        

      

      

        
04

        
Modular Code Design

        
Code the system to auto-discover new tools using a shared template.

        
    
          
  • Easy Growth: Add a new tool with just one simple file.
    • 
          
  • Better Care: Keeps "engine" logic separate from UI "buttons".

Icon Finding & Setup


    

      

        
01

        
Search Strategy

      

      
Visit Flaticon.com and search with English keywords (e.g., CAD, drawing, tool).

      
  • Stick to a consistent style — Flat, Line, or Solid — to maintain UI harmony across the add-in.

    


    

      

        
02

        
Color Customization

      

      
Login to Flaticon and click the Edit (pencil) icon to adjust colors.

      
  • Ensure colors match the official tool branding or the Inventor theme for professional integration.

    


    

      

        
03

        
Sizing (Crucial)

      

      
Click the dropdown arrow next to PNG. Export exactly two sizes:

      
    
        
  • 16px — For Small/Standard ribbon icons
    • 
        
  • 32px — For Large ribbon buttons
    • 
      

    


    

      

        
04

        
Download & Resource

      

      
Download as PNG to preserve transparency.

      
  • Move files into your project's Resource folder to keep the assembly self-contained and clean.
