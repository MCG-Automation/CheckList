Tổng quan

MacGregor QA/QC Checklist là gì?

Add-in tích hợp trực tiếp vào AutoCAD, giúp kỹ sư thực hiện quy trình kiểm tra chất lượng (QA/QC) bản vẽ một cách có hệ thống trước khi phát hành lên Vault. Thay vì dùng checklist giấy hoặc file Excel rời, mọi dữ liệu kiểm tra được nhúng trực tiếp vào file .dwg.

Dữ liệu theo bản vẽ — gửi file DWG cho bất kỳ ai, họ cũng thấy trạng thái QA/QC đầy đủ khi mở tool.

⚠ Khi nào BẮT BUỘC dùng Checklist?
Checklist chỉ bắt buộc với các bản vẽ Main — bản vẽ cuối cần Release lên Vault. Các bản vẽ Global (WIP / phối hợp nhóm) có thể bỏ qua.
📋

Checklist theo loại bản vẽ

Chọn Layout/Interface, Mechanical hoặc Structure — tool tự load đúng danh sách kiểm tra từ template Excel.

💾

Dữ liệu nhúng vào DWG

Tiến độ tích checklist được lưu ẩn bên trong file DWG. Mở lại bất kỳ lúc nào, dữ liệu vẫn còn nguyên.

Quy trình sử dụng

01

Mở Checklist Palette

Trong Command Line của AutoCAD, gõ lệnh:

Command: MCG_Checklist

Panel MCG Tools xuất hiện ở bên phải màn hình AutoCAD. Gọi lại lệnh khi panel đang mở sẽ hiển thị lại panel (không tạo thêm panel mới).

Bản vẽ đã từng dùng Checklist? Panel tự động khôi phục đúng loại checklist và trạng thái đã tích từ lần trước — bạn không cần chọn lại loại bản vẽ.
02

Chọn loại bản vẽ

Nếu đây là bản vẽ mới (lần đầu dùng Checklist), panel hiển thị "Select a drawing type above". Mở dropdown ở góc trên phải và chọn loại bản vẽ phù hợp:

  • Layout / Interface — bản vẽ bố trí tổng thể, giao diện
  • Mechanical — chi tiết cơ khí
  • Structure — kết cấu thép, panel

Sau khi chọn, tool đồng bộ template Excel mới nhất từ Vault Server và tải checklist tương ứng (overlay loading xuất hiện trong vài giây).

Nếu không có Vault, có thể bấm nút ... để chọn file Excel thủ công từ máy tính.

03

Thực hiện kiểm tra

  • Tick vào ô ✓ của từng câu hỏi khi đã kiểm tra xong.
  • Nếu một mục không áp dụng cho bản vẽ này → tick cột N/A.
  • Cả hai loại (✓ và N/A) đều được tính vào tiến độ.
  • Thanh Progress Bar cập nhật phần trăm hoàn thành ngay lập tức.
  • Dữ liệu được tự động lưu sau 500ms — không cần nhấn Save thủ công.
Có thể dừng giữa chừng không? Hoàn toàn được. Đóng AutoCAD hay tắt máy bất kỳ lúc nào — mở lại bản vẽ hôm sau, tool khôi phục đúng trạng thái đã tích.
04

Thêm câu hỏi tùy chỉnh (Custom Item)

Nếu checklist chuẩn chưa bao gồm một tiêu chí đặc thù cho dự án, bạn có thể thêm thủ công:

  1. Gõ nội dung câu hỏi vào ô "Enter custom item content here..." ở cuối panel.
  2. Bấm nút ADD NEW.
  3. Câu hỏi tự chọn xuất hiện ở cuối danh sách, có nút ✕ để xóa nếu cần.
Custom Items được lưu cùng bản vẽ. Khi mở lại DWG, các câu hỏi tự chỉnh vẫn xuất hiện đầy đủ ở cuối danh sách.
05

Ký duyệt — APPROVE

Khi đã kiểm tra xong (hoặc đánh N/A cho các mục không áp dụng), bấm nút APPROVE màu xanh lá ở cuối panel:

  • Nếu còn mục chưa tích → hộp thoại cảnh báo xuất hiện, xác nhận để tiếp tục.
  • Hệ thống tự động ghi tên Windows user và thời gian duyệt vào bản vẽ DWG.
  • Trạng thái chuyển sang APPROVED (màu xanh lá), toàn bộ danh sách bị khoá.
Bản vẽ đã APPROVED → sẵn sàng Check-in lên Vault để xin Release!
06

Mở lại để kiểm tra — RE-OPEN

Bản vẽ đã Approve nhưng cần sửa lại? Bấm nút RE-OPEN màu đỏ:

  • Trạng thái trở về PENDING, checklist được mở khoá để chỉnh sửa.
  • Thông tin người duyệt và thời gian bị xoá.
  • Các mục đã tích trước đó không bị xóa — chỉ mở khoá để bạn điều chỉnh thêm.

Sau khi kiểm tra lại xong, bấm APPROVE lần 2 để chốt lại.

Khi RE-OPEN, bản vẽ không còn ở trạng thái APPROVED — nếu đã Check-in Vault thì cần check lại sau khi Approve lần 2.

Tính năng Chi tiết

Dropdown Chọn loại bản vẽ

Mỗi loại bản vẽ tương ứng với một file Excel template riêng được lưu trên Vault:

Layout / Interface → Temp Checklist - LayoutInterface.xlsx
Mechanical → Temp Checklist - Mechanical.xlsx
Structure → Temp Checklist - Structure.xlsx

Khi chọn một loại, tool thực hiện: (1) kết nối Vault, (2) tải Excel mới nhất, (3) merge với dữ liệu đã tích trước đó (Carry-over). Tên loại bản vẽ đã chọn được ghi vào DWG để tự khôi phục lần sau.

Trạng thái đồng bộ Vault (Sync Status)

Dòng nhỏ dưới tiêu đề "MCG CHECKLIST" hiển thị nguồn gốc file Excel đang dùng:

✓  Vault ✔ 09:35 — Template mới nhất tải từ Vault lúc 09:35
⚠  Vault offline — Login failed — Dùng file local cũ
📁  Local file — File Excel chọn thủ công
Vault offline không ảnh hưởng đến checklist — tool dùng file Excel từ lần tải cuối (local cache). Bạn vẫn tick, approve bình thường.

Thanh Tiến độ (Progress Bar)

Công thức tính %

% = (Số mục ✓ + Số mục N/A) ÷ Tổng số mục × 100

Cả ✓ lẫn N/A đều tính vào tiến độ hoàn thành.

Hiển thị

  • Thanh màu xanh ■ tăng theo thời gian thực.
  • Phần trăm hiển thị bên phải: 72%
  • Cập nhật ngay khi bạn tick hoặc bỏ tick.

Auto-save Debounce 500ms

Mỗi lần tick ô, tick N/A, thêm/xóa Custom Item — tool bắt đầu đếm ngược 500ms. Sau 500ms không có thêm thao tác nào, dữ liệu được lưu tự động vào:

1. JSON local cache
%APPDATA%\MCG_CheckList\
2. DWG file
Nhúng vào bản vẽ đang mở

Debounce có nghĩa là: nếu bạn tick liên tục 10 ô trong 2 giây, hệ thống chỉ ghi đĩa 1 lần duy nhất (sau khi bạn dừng tay 500ms), không bị ghi liên tục.

Carry-over — Giữ tiến độ khi template Excel thay đổi

Khi Vault cập nhật phiên bản mới của file Excel template (thêm/sửa câu hỏi), tool thực hiện merge thông minh giữa template mới và tiến độ đã tích:

  • Câu hỏi nào giống nhau (trùng nội dung) → giữ nguyên trạng thái đã tích.
  • Câu hỏi mới thêm vào template → xuất hiện chưa tích.
  • Câu hỏi đã bị xoá khỏi template → không hiển thị nữa.
  • Custom Items → luôn được giữ lại, xuất hiện ở cuối danh sách.

Dữ liệu được lưu ở đâu trong file DWG?

Tool dùng Named Object Dictionary của AutoCAD — vùng dữ liệu ẩn chuẩn của định dạng DWG, không ảnh hưởng đến hình học bản vẽ:

DBDictionary "MACGREGOR_QA_SYSTEM"
└── XRecord "CHECKLIST_DATA"
└── DxfCode.Text × N chunks ← JSON được chia nhỏ, 250 ký tự/chunk

Dữ liệu JSON của toàn bộ Checklist (danh sách câu hỏi, trạng thái tick, người duyệt...) được chia thành các đoạn 250 ký tự và lưu vào XRecord. Khi đọc lại, các đoạn được ghép lại và parse thành object.

Cài đặt

01

Copy file DLL vào thư mục nguồn

Lấy toàn bộ file MCG_*.dll (từ bản build hoặc thư mục Release do IT cung cấp) và copy vào:

C:\CustomTools\Autocad\

File Install_AutoLoadCadAddin.bat cũng phải nằm trong thư mục này.

02

Chạy Batch file cài đặt

Chuột phải vào Install_AutoLoadCadAddin.bat → chọn "Run as Administrator".

Script tự động:

  • Tạo thư mục %PROGRAMDATA%\Autodesk\ApplicationPlugins\MCG_Plugin.bundle\Contents\
  • Copy tất cả MCG_*.dll vào thư mục Contents
  • Tạo file PackageContents.xml cho AutoCAD nhận diện plugin
Phải chạy với quyền Administrator. Nếu không, script báo lỗi "Cannot create directory" và thoát.
03

Khởi động lại AutoCAD

Khởi động lại AutoCAD để plugin được nạp tự động. Kiểm tra bằng cách gõ lệnh:

Command: MCG_Checklist

Nếu Panel MCG Tools xuất hiện → cài đặt thành công.

Cập nhật phiên bản mới

Khi có phiên bản plugin mới:

  1. Đóng AutoCAD.
  2. Copy file DLL mới vào C:\CustomTools\Autocad\ (ghi đè file cũ).
  3. Chạy lại Install_AutoLoadCadAddin.bat với quyền Administrator.
  4. Mở lại AutoCAD.
Dữ liệu checklist trong các file DWG cũ không bị ảnh hưởng khi cập nhật plugin — dữ liệu nằm trong bản vẽ, không phải trong plugin.

Nạp thủ công bằng NETLOAD (chỉ dùng để test)

Dùng khi cần test nhanh mà không muốn cài đặt toàn bộ:

  1. Mở AutoCAD và mở một bản vẽ bất kỳ.
  2. Gõ lệnh NETLOAD trên Command Line.
  3. Duyệt đến file MCG_Checklist.dll và bấm Open.
  4. Nếu có cảnh báo bảo mật → chọn Always Load.
  5. Gõ MCG_Checklist để mở panel.
NETLOAD chỉ nạp plugin cho phiên AutoCAD đang mở. Đóng và mở lại AutoCAD thì plugin mất — phải NETLOAD lại. Dùng batch file cài đặt để plugin tự nạp mỗi lần khởi động.

FAQ & Khắc phục sự cố

Q: Gõ MCG_Checklist nhưng AutoCAD báo "Unknown command"? ▶
Plugin chưa được nạp vào phiên AutoCAD này. Kiểm tra theo thứ tự:
  1. Nếu dùng auto-load: kiểm tra thư mục %PROGRAMDATA%\Autodesk\ApplicationPlugins\MCG_Plugin.bundle\Contents\ có chứa file MCG_Checklist*.dll không. Nếu không — chạy lại Install_AutoLoadCadAddin.bat với quyền Administrator, sau đó khởi động lại AutoCAD.
  2. Nếu dùng NETLOAD: gõ lại lệnh NETLOAD và chọn đúng file DLL.
  3. Vẫn lỗi: mở DebugView (Sysinternals) hoặc VS Output window, tìm log có prefix [PaletteManager] để xem chi tiết lỗi.
Q: Dropdown chọn loại bản vẽ bị mờ (disabled), không bấm được? ▶
Dropdown bị khoá khi đang tải dữ liệu (overlay loading đang hiển thị). Hãy chờ overlay biến mất, thường dưới 10 giây khi kết nối Vault bình thường. Nếu sau 30 giây vẫn loading → kiểm tra kết nối mạng đến Vault Server. Sau khi Vault timeout, tool sẽ fallback về file local và tự mở khoá dropdown.
Q: Mở lại file DWG nhưng checklist bắt đầu từ đầu, không thấy dữ liệu cũ? ▶
Có thể xảy ra do một trong các nguyên nhân sau:
  • File DWG được lưu bởi phần mềm khác (AutoCAD LT, eDrawings, trình xem DWG...) có thể strip Named Object Dictionary → dữ liệu bị mất. Luôn lưu bằng AutoCAD chính thức.
  • Đóng AutoCAD quá nhanh sau khi tick: debounce timer 500ms chưa kịp lưu. Trong tương lai hãy chờ 1 giây sau thao tác cuối trước khi đóng.
  • File DWG từ trước khi cài plugin: lần đầu mở bằng plugin, cần chọn lại loại bản vẽ từ dropdown. Sau đó dữ liệu được lưu và tự khôi phục từ lần sau.
Kiểm tra JSON cache tại %APPDATA%\MCG_CheckList\ — nếu file JSON vẫn còn, tool sẽ đọc từ đó khi không tìm thấy dữ liệu trong DWG.
Q: Tắt máy giữa chừng, liệu có mất dữ liệu checklist không? ▶
Rủi ro rất thấp. Auto-save debounce 500ms đảm bảo dữ liệu được lưu vào JSON cache mỗi khi bạn dừng thao tác 500ms — xảy ra trước khi lưu vào DWG. Khi mở lại, tool đọc DWG trước; nếu DWG chưa có (ví dụ chưa kịp lưu lần cuối) thì đọc từ JSON cache. Dữ liệu chỉ mất hoàn toàn nếu máy tắt trong đúng khoảnh khắc đang ghi cả DWG lẫn JSON — trường hợp cực kỳ hiếm.
Q: Panel tự load checklist khi mở tool dù tôi chưa chọn gì? ▶
Đây là tính năng tự động khôi phục: nếu file DWG đang mở đã có dữ liệu checklist từ trước (lưu trong Named Object Dictionary), tool tự nhận ra loại bản vẽ đã chọn và tải lại đúng checklist đó — không cần bạn chọn lại dropdown. Nếu muốn chuyển sang loại checklist khác, chọn từ dropdown — tool sẽ merge tiến độ cũ với template mới (Carry-over).
Q: Thông báo "Vault offline" xuất hiện — checklist có dùng được không? ▶
Có, vẫn dùng bình thường. Vault offline chỉ nghĩa là tool không tải được phiên bản Excel template mới nhất — tool sẽ dùng file Excel đã tải từ lần trước trong local cache. Bạn vẫn tick, approve, lưu dữ liệu hoàn toàn bình thường. Khi kết nối Vault được phục hồi, lần mở tool tiếp theo sẽ tự đồng bộ template mới nhất.
Q: Làm thế nào biết bản vẽ nào đã được Approve hay chưa? ▶
Mở bản vẽ trong AutoCAD → gõ lệnh MCG_Checklist. Phần DOCUMENT STATUS ở cuối panel sẽ hiển thị:
  • ✅ APPROVED — [Tên user] · [Thời gian duyệt]
  • ⏳ PENDING — Chưa hoàn thành kiểm tra
Nếu bản vẽ chưa bao giờ dùng checklist → panel hiển thị "Select a drawing type above" và không có status.
Q: Tôi muốn đổi loại checklist cho bản vẽ (ví dụ từ Structure sang Mechanical)? ▶
Chọn lại từ dropdown — tool sẽ tải template Mechanical mới và merge (Carry-over) với những gì bạn đã tích trước đó. Các mục trùng nội dung giữa 2 template sẽ giữ trạng thái tick; các mục mới trong Mechanical sẽ hiện chưa tích. Custom Items được giữ nguyên. Tên loại mới được lưu vào DWG để tự khôi phục lần sau.
Q: Checklist template Excel được cập nhật phiên bản mới — dữ liệu đã tích có bị mất không? ▶
Không mất. Tính năng Carry-over tự động merge template mới với dữ liệu cũ:
  • Câu hỏi nào vẫn còn trong template mới → giữ nguyên trạng thái đã tích.
  • Câu hỏi mới thêm vào template → xuất hiện chưa tích.
  • Custom Items → luôn được giữ lại.
Bạn chỉ cần tích thêm các câu hỏi mới — không phải làm lại từ đầu.
Q: Sau khi Approve, tôi có thể xem lại checklist mà không bị mở khoá không? ▶
Có. Khi bản vẽ ở trạng thái APPROVED, panel vẫn hiển thị toàn bộ danh sách câu hỏi với trạng thái tick — chỉ là các checkbox bị khoá không bấm được. Bạn có thể cuộn xem tất cả, xem tên người duyệt và thời gian duyệt ở cuối panel. Chỉ khi bấm RE-OPEN mới mở khoá để chỉnh sửa.

Vault Integration

Tích hợp hiện tại — Đồng bộ Template Excel từ Vault

Mỗi lần chọn loại bản vẽ từ dropdown, tool tự động kết nối Vault và tải file Excel template mới nhất. Quy trình:

1
Đọc LoginHistory.xml (%APPDATA%\Autodesk\VaultCommon\) để lấy tên Vault Server đang dùng — không cần nhập tay.
2
Đăng nhập Vault với AutoLoginMode = RestoreAndExecute — dùng lại session Vault đang active, không hiện hộp thoại đăng nhập.
3
Download file Excel từ đường dẫn Vault được cấu hình trong appsettings.json → lưu vào Working Folder local.
4
Bóc tách nội dung Excel, merge với tiến độ cũ (Carry-over), hiển thị checklist.
Nếu Vault không kết nối được, tool fallback về file Excel đã download lần trước trong local cache. Checklist vẫn hoạt động bình thường.
ROADMAP — Chưa triển khai

Tự động hóa 100%: Chặn Release nếu chưa Approve

Mục tiêu dài hạn: Vault tự động từ chối chuyển bản vẽ sang trạng thái Released nếu kỹ sư chưa Approve checklist. Lộ trình triển khai gồm 4 bước:

BƯỚC 0 — Dev

Thêm Bridge Drawing Property trong Add-in

Khi user bấm Approve, add-in ghi thêm Custom Drawing Property MacGregor_QA = "APPROVED" vào DWG (qua SummaryInfo hoặc DwgProperty). Vault đọc được property này. Khi RE-OPEN → xoá property này.

BƯỚC 1 — Vault Admin

Tạo Property trên Vault

Tools → Administration → Vault Settings → Behaviors → Properties → Tạo property QA Status (loại Text).

BƯỚC 2 — Vault Admin

Ánh xạ từ DWG Property → Vault Property

Tab Mapping của property: Entity = File, Provider = AutoCAD, File Property = MacGregor_QA, Direction = File → Vault.

BƯỚC 3 — Vault Admin

Thêm điều kiện vào Lifecycle

Vault Settings → Behaviors → Lifecycles → Sửa lifecycle bản vẽ Main → Transition Work in Progress → Released → Tab Criteria: thêm điều kiện QA Status is APPROVED.

Kết quả sau 4 bước: Khi người duyệt cố Release bản vẽ chưa có QA Approve, Vault hiện thông báo lỗi và từ chối — quy trình QA/QC hoàn toàn tự động, không thể bỏ qua.
