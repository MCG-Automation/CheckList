Overview

What is the MacGregor QA/QC Checklist?

An add-in integrated directly into AutoCAD, enabling engineers to carry out a structured Quality Assurance / Quality Control review of drawings before releasing them to Vault. Instead of paper checklists or standalone Excel files, all inspection data is embedded directly inside the .dwg file.

Data travels with the drawing — share the DWG with anyone and they can open the tool to see the full QA/QC status.

⚠ When is the Checklist mandatory?
The checklist is required only for Main drawings — the final drawing that needs to be Released to Vault. Global drawings (WIP / team collaboration) may skip this step.

How to Use

01

Open the Checklist Palette

In the AutoCAD Command Line, type:

Command: MCG_Checklist

The MCG Tools palette panel appears on the right side of the AutoCAD screen. Running the command again while the panel is open simply brings it back into focus — no duplicate panel is created.

Drawing already has a Checklist? The panel automatically restores the correct checklist type and all previously ticked items — no need to re-select the drawing type from the dropdown.
02

Select the Drawing Type

If this is a new drawing (first time using the Checklist), the panel shows "Select a drawing type above". Open the dropdown in the top-right corner and choose the appropriate type:

  • Layout / Interface — general arrangement, interface drawings
  • Mechanical — mechanical detail drawings
  • Structure — structural steel, panel drawings

After selecting, the tool syncs the latest Excel template from Vault and loads the corresponding checklist (a loading overlay appears for a few seconds).

If Vault is unavailable, click the ... button to manually browse for an Excel file on your computer.

03

Perform the Inspection

  • Tick the ✓ checkbox next to each item once you have verified it.
  • If an item does not apply to this drawing → tick the N/A column instead.
  • Both ✓ and N/A count toward the overall progress.
  • The Progress Bar updates the completion percentage instantly.
  • Data is auto-saved after 500 ms — no manual Save button required.
Can I stop midway? Absolutely. Close AutoCAD or shut down the computer at any point — reopen the drawing the next day and the tool restores exactly where you left off.
04

Add a Custom Check Item

If the standard checklist does not cover a project-specific criterion, you can add one manually:

  1. Type the item content into the "Enter custom item content here..." field at the bottom of the panel.
  2. Click ADD NEW.
  3. The custom item appears at the end of the list with a ✕ button to delete it if needed.
Custom Items are saved inside the DWG. When the drawing is reopened, all custom items reappear at the end of the list.
05

Sign Off — APPROVE

Once the inspection is complete (or N/A has been set for non-applicable items), click the green APPROVE button at the bottom of the panel:

  • If unchecked items remain → a warning dialog appears; confirm to proceed anyway.
  • The system automatically records the Windows username and approval timestamp inside the DWG.
  • Status changes to APPROVED (green), and the entire checklist locks.
Drawing is APPROVED → ready to check in to Vault and request Release!
06

Unlock for Re-inspection — RE-OPEN

Already approved but need to make changes? Click the red RE-OPEN button:

  • Status reverts to PENDING and the checklist is unlocked for editing.
  • The approver name and timestamp are cleared.
  • Previously ticked items are not removed — the checklist is simply unlocked for further adjustments.

After re-inspecting, click APPROVE again to finalise.

After RE-OPEN the drawing is no longer in APPROVED state. If it has already been checked in to Vault, a new check-in will be required after the second Approval.

Feature Details

Drawing Type Dropdown

Each drawing type maps to a dedicated Excel template file stored on Vault:

Layout / Interface → Temp Checklist - LayoutInterface.xlsx
Mechanical → Temp Checklist - Mechanical.xlsx
Structure → Temp Checklist - Structure.xlsx

When a type is selected, the tool: (1) connects to Vault, (2) downloads the latest Excel, (3) merges with any previously ticked data (Carry-over). The selected drawing type name is written into the DWG so it can be auto-restored next time.

Vault Sync Status

A small line below the "MCG CHECKLIST" heading shows the origin of the Excel template currently in use:

✓  Vault ✔ 09:35 — Latest template downloaded from Vault at 09:35
⚠  Vault offline — Login failed — Using previously cached local file
📁  Local file — Excel file selected manually
Vault being offline does not affect the checklist — the tool uses the Excel file from the last successful download. Ticking and approving work as normal.

Progress Bar

Calculation

% = (Items ✓ + Items N/A) ÷ Total items × 100

Both ✓ and N/A count toward completion.

Display

  • Blue bar ■ grows in real time.
  • Percentage shown on the right: 72%
  • Updates immediately on every tick or un-tick.

Auto-save Debounce — 500 ms

Every time you tick, un-tick, mark N/A, or add/delete a Custom Item, a 500 ms countdown starts. If no further action occurs within that window, data is automatically saved to:

1. JSON local cache
%APPDATA%\MCGCadPlugin\
2. DWG file
Embedded in the open drawing

Debounce means: if you tick 10 items in 2 seconds, the system writes to disk only once — after you pause for 500 ms — rather than on every individual tick.

Carry-over — Preserving Progress When the Excel Template Updates

When Vault publishes a new version of the Excel template (questions added or modified), the tool performs an intelligent merge between the new template and existing ticked data:

  • Questions that still exist in the new template (matching content) → tick state is preserved.
  • Questions newly added to the template → appear unticked.
  • Questions removed from the template → no longer shown.
  • Custom Items → always retained and shown at the end of the list.

Approval Workflow

PENDING

→ APPROVE →
APPROVED

→ RE-OPEN →
PENDING

On APPROVE: writes Windows username (Environment.UserName) and current timestamp into the DWG. All checkboxes and the ADD NEW field are locked.

On RE-OPEN: clears the approver name and timestamp, resets status to PENDING, and unlocks all checkboxes. Previously ticked items are not cleared.

DOCUMENT STATUS examples

✅ APPROVED — John Smith · 15/06/2025 09:42
⏳ PENDING — Inspection not yet complete

Where Is the Data Stored Inside the DWG?

The tool uses AutoCAD's Named Object Dictionary — a standard hidden data area of the DWG format that does not affect drawing geometry:

DBDictionary "MACGREGOR_QA_SYSTEM"
└── XRecord "CHECKLIST_DATA"
└── DxfCode.Text × N chunks ← JSON split into 250-character chunks

The full checklist JSON (questions, tick states, approver info…) is split into 250-character segments and stored in the XRecord. On read, the segments are reassembled and parsed back into an object.

Installation

01

Copy DLL files to the source folder

Obtain all MCG_*.dll files (from the build output or the Release folder provided by IT) and copy them to:

C:\CustomTools\Autocad\

The Install_AutoLoadCadAddin.bat file must also be placed in this folder.

02

Run the installation batch file

Right-click Install_AutoLoadCadAddin.bat → select "Run as Administrator".

The script automatically:

  • Creates %PROGRAMDATA%\Autodesk\ApplicationPlugins\MCG_Plugin.bundle\Contents\
  • Copies all MCG_*.dll files into the Contents folder
  • Generates PackageContents.xml so AutoCAD can discover the plugin
Must be run as Administrator. Without elevated privileges the script reports "Cannot create directory" and exits immediately.
03

Restart AutoCAD

Restart AutoCAD so the plugin is loaded automatically. Verify by typing:

Command: MCG_Checklist

If the MCG Tools panel appears → installation was successful.

Updating to a New Version

When a new plugin version is released:

  1. Close AutoCAD.
  2. Copy the new DLL files into C:\CustomTools\Autocad\, overwriting the old ones.
  3. Re-run Install_AutoLoadCadAddin.bat as Administrator.
  4. Reopen AutoCAD.
Checklist data stored in existing DWG files is not affected by a plugin update — data lives in the drawings, not in the plugin itself.

Manual Load via NETLOAD (for testing only)

Use this method for a quick test without going through the full installation:

  1. Open AutoCAD and open any drawing.
  2. Type NETLOAD in the Command Line.
  3. Browse to MCG_Checklist.dll and click Open.
  4. If a security warning appears → choose Always Load.
  5. Type MCG_Checklist to open the panel.
NETLOAD only loads the plugin for the current AutoCAD session. Closing and reopening AutoCAD unloads it — you must NETLOAD again. Use the batch-file installation to have the plugin load automatically on every startup.

FAQ & Troubleshooting

Q: I typed MCG_Checklist but AutoCAD says "Unknown command"? ▶
The plugin has not been loaded into this AutoCAD session. Check in order:
  1. If using auto-load: verify that %PROGRAMDATA%\Autodesk\ApplicationPlugins\MCG_Plugin.bundle\Contents\ contains a MCG_Checklist*.dll file. If not — re-run Install_AutoLoadCadAddin.bat as Administrator, then restart AutoCAD.
  2. If using NETLOAD: run the NETLOAD command again and select the correct DLL.
  3. Still failing: open DebugView (Sysinternals) or the VS Output window and look for log entries with the prefix [PaletteManager] for detailed error information.
Q: The drawing type dropdown is greyed out and cannot be clicked? ▶
The dropdown is disabled while data is loading (the loading overlay is visible). Wait for the overlay to disappear — this normally takes under 10 seconds on a healthy Vault connection. If the loading spinner persists after 30 seconds, check your network connection to the Vault server. Once the Vault connection times out the tool falls back to the local cache and the dropdown automatically re-enables.
Q: I reopened a DWG but the checklist starts from scratch — previous data is gone? ▶
This can happen for one of the following reasons:
  • DWG saved by third-party software (AutoCAD LT, eDrawings, DWG viewers…) — these applications can strip the Named Object Dictionary, removing all embedded data. Always save using full AutoCAD.
  • AutoCAD closed too quickly after the last tick: the 500 ms debounce timer may not have had time to write. Going forward, wait about 1 second after your last action before closing.
  • DWG predates the plugin: the first time you open a drawing with the plugin, you must select the drawing type from the dropdown. After that, data is saved and auto-restored on every subsequent open.
Check the JSON cache at %APPDATA%\MCGCadPlugin\ — if the JSON file is still present, the tool will read from it when no data is found in the DWG.
Q: My computer shut down unexpectedly — will I lose checklist data? ▶
Risk is very low. The 500 ms auto-save debounce ensures data is written to the JSON cache every time you pause for 500 ms — this happens before the DWG write. On reopen, the tool reads the DWG first; if the DWG write was incomplete (e.g. power cut mid-save) it falls back to the JSON cache. Data is only completely lost if the machine shuts down at the exact moment both the DWG and the JSON are being written simultaneously — an extremely rare scenario.
Q: The panel loads a checklist automatically when I open the tool — I didn't select anything? ▶
This is the auto-restore feature: if the currently open DWG already contains checklist data (stored in its Named Object Dictionary), the tool detects which drawing type was previously selected and reloads the exact same checklist — no dropdown selection required. To switch to a different checklist type, simply choose a new option from the dropdown. The tool will carry over your existing progress into the new template (Carry-over merge).
Q: The "Vault offline" message appears — can I still use the checklist? ▶
Yes, everything works normally. "Vault offline" only means the tool could not download the latest Excel template — it falls back to the version downloaded during the previous session (local cache). Ticking, approving, and saving all function as usual. The next time the tool opens with an active Vault connection, the template will be re-synced automatically.
Q: How do I know whether a drawing has already been Approved? ▶
Open the drawing in AutoCAD → type MCG_Checklist. The DOCUMENT STATUS section at the bottom of the panel shows:
  • ✅ APPROVED — [Username] · [Approval date & time]
  • ⏳ PENDING — Inspection not yet complete
If the drawing has never used the checklist, the panel shows "Select a drawing type above" with no status indicator.
Q: I want to switch the checklist type for a drawing (e.g. from Structure to Mechanical)? ▶
Simply select the new type from the dropdown. The tool loads the Mechanical template and merges (Carry-over) it with your existing ticked data. Items whose content matches between the two templates retain their tick state; new items in the Mechanical template appear unticked. Custom Items are always preserved. The new drawing type name is saved to the DWG for auto-restore next time.
Q: The Excel checklist template was updated — will my ticked data be lost? ▶
No. The Carry-over feature automatically merges the new template with existing data:
  • Questions still present in the new template → tick state preserved.
  • Newly added questions → appear unticked.
  • Custom Items → always retained.
You only need to tick the new questions — there is no need to redo the entire checklist.
Q: After Approving, can I view the checklist without unlocking it? ▶
Yes. When a drawing is in APPROVED state, the panel still displays the full list of questions with all tick states visible — the checkboxes are simply locked and cannot be clicked. You can scroll through everything and see the approver name and approval timestamp at the bottom. Only clicking RE-OPEN will unlock the checklist for editing.

Vault Integration

Current Integration — Excel Template Sync from Vault

Every time a drawing type is selected from the dropdown, the tool automatically connects to Vault and downloads the latest Excel template. The process:

1
Reads LoginHistory.xml (%APPDATA%\Autodesk\VaultCommon\) to detect the active Vault server — no manual entry required.
2
Logs in to Vault with AutoLoginMode = RestoreAndExecute — reuses the active Vault session silently, with no login dialog shown to the user.
3
Downloads the Excel file from the Vault path configured in appsettings.json → saves it to the local working folder.
4
Parses the Excel content, merges with previous progress (Carry-over), and renders the checklist.
If Vault is unreachable, the tool falls back to the Excel file downloaded during the last successful sync. The checklist continues to work normally.
ROADMAP — Not yet implemented

Full Automation: Block Release Until Approved

Long-term goal: Vault automatically rejects any attempt to transition a drawing to Released status if the engineer has not yet Approved the checklist. The rollout plan has 4 steps:

STEP 0 — Development

Add a Bridge Drawing Property in the Add-in

When the user clicks Approve, the add-in writes a Custom Drawing Property MacGregor_QA = "APPROVED" to the DWG (via SummaryInfo or DwgProperty). Vault can read this property directly. On RE-OPEN → the property is cleared.

STEP 1 — Vault Admin

Create a Property in Vault

Tools → Administration → Vault Settings → Behaviors → Properties → Create a property named QA Status (type: Text).

STEP 2 — Vault Admin

Map DWG Property → Vault Property

In the Mapping tab of the property: Entity = File, Provider = AutoCAD, File Property = MacGregor_QA, Direction = File → Vault.

STEP 3 — Vault Admin

Add a Lifecycle Criterion

Vault Settings → Behaviors → Lifecycles → Edit the Main drawing lifecycle → select the Work in Progress → Released transition → Criteria tab: add condition QA Status is APPROVED.

Result after all 4 steps: When a reviewer tries to Release a drawing that has not been QA-Approved, Vault displays an error and refuses — the QA/QC process becomes fully automated and impossible to bypass.
